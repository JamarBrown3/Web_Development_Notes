Javascript makeup

JavaScript applications consist of statements with an appropriate syntax. A single statement may span multiple lines. Multiple statements may occur on a single line if each statement is separated by a semicolon. This isn't a keyword, but a group of keywords. In HTML, JavaScript statements are "instructions" to be "executed" by the web browser.

Javascript statements is usually made up of:

  • Values
  • Literals
  • Variables
  • Operators
  • Expressions
  • Keywords

In JavaScript, values are the pieces of data that the program works with. They can represent anything such as text, numbers, true/false information, or more complex data structures. Everything the program processes, stores, or outputs is considered a value.

Literals are fixed values that are written directly into a program. They do not change unless the programmer edits them. Examples include written text, numbers, and true/false values that appear exactly as they are used in the code.

Variables are containers used to store values in a JavaScript program. Unlike literals, the data inside a variable can change while the program is running, allowing JavaScript to work with dynamic and flexible information.

Operators are symbols or words that tell JavaScript to perform specific actions on values or variables. They are used for tasks such as performing math, comparing values, or assigning data to variables.

Keywords are special reserved words that have specific meanings in JavaScript and cannot be used for anything else. They tell the programming language to perform particular tasks, such as declaring a variable or defining a function.

Important Data Types

  • document: When a member returns an object of type document (e.g., the ownerDocument property of an element returns the document to which it belongs), this object is the root document object itself.
  • element: element refers to an element or a node of type element returned by a member of the DOM API.

    Rather than saying, for example, that the document.createElement() method returns an object reference to a node, we just say that this method returns the element that has just been created in the DOM. element objects implement the DOM Element interface and also the more basic Node interface, both of which are included together in this reference.
  • nodeList: A nodeList is a collection of elements similar to an array, often returned by DOM methods like document.getElementsByTagName(). You can access items in a nodeList using either the item() method or standard array-style indexing. For example, list.item(1) and list[1] both retrieve the second element in the list. These two approaches work the same way and are interchangeable.
  • Attribute: When an attribute is returned in the DOM—for example, through the createAttribute() method—it is treated as an object with its own small interface. In the DOM, attributes are actually considered nodes, similar to elements, although in practice you rarely need to work with them directly as nodes.
  • namedNodeMap: A namedNodeMap is similar to an array, but its items can be accessed by either a name or an index. Accessing by index is mainly for convenience when you need to loop through the items, since the items are not stored in any specific order. A namedNodeMap provides an item() method to access elements, and it also allows you to add or remove items from the collection.

    JavaScript Cookies

    • Cookies were originally invented by Netscape to give 'memory' to web servers and browsers.
    • The HTTP protocol is state-less, which means that once the server has sent a page to a browser requesting it, it doesn't remember a thing about it.
    • So if you come to the same web page a second, third, hundredth or millionth time, the server once again considers it the very first time you ever came there.

    What are cookies?

    • Small text data (<4 KB) stored on the user's computer cookies that exceed this length are trimmed to fit.
    • Remember information about the user across visits (e.g., username).
    • Default: deleted when browser closes; can set expiration to keep longer.
    • Accessed via document.cookie.
    • Stored as name=value pairs. Only the site that set the cookie can read it.

    Why use cookies?

    • User Recognition: Identify returning visitors
    • Session Managemen: Remember login sessions, passwords (encrypted)
    • Personalization: Save user preferences (themes, settings)
    • Tracking: Monitor site navigation, page views, ad relevance
    • E-commerce: Maintain shopping cart items between visits

    How it works?

    1. Your server sends some data to the visitor's browser in the form of a cookie.
    2. The browser may accept the cookie. If it does, it is stored as a plain text record on the visitor's hard drive.
    3. Now, when the visitor arrives at another page on your site, the browser sends the same cookie to the server for retrieval.
    4. Once retrieved, your server knows/remembers what was stored earlier.

    During the browsing session browser stores the cookies in memory, at the time of quitting they goto the file called cookies.txt. Different browser store cookies files in a different location in the disk.

    Attributes of a Cookie

    • Name: Alphanumeric string, no spaces or commas (underscores allowed). Case-insensitive. Must be a single string with no commas, semicolons, or whitespace.
    • Value: Data to store; cannot be undefined or null, but can be empty.
    • Expires: Sets when the cookie expires (default = end of session). Format: DD-Mon-YY HH:MM:SS UTC.
    • Path: Limits cookie to a specific site path (default = path where created).
    • Domain: Restricts cookie to a particular domain (default = current domain).
    • Secure: true = cookie sent only over HTTPS (default = false).
    • Accessed via document.cookie.
    • Can read, create, modify, and delete cookies for the current webpage.

    JavaScript API Call

    The fetch() function initiates a request by taking the URL of the resource and returns a Promise that resolves to a Response object. The .then() method is used to handle this response.

    After receiving the response, it is important to check whether the request was successful using the response.ok property, which is true for HTTP status codes in the 200-299 range. If the response is not successful, an error should be thrown.

    The response body must be parsed to access the actual data. Common parsing methods include response.json() for JSON data, response.text() for plain text, and response.blob() for binary data. These methods return Promises that resolve to usable JavaScript values.

    Once the data is parsed, another .then() block is used to work with the returned data inside the application.

    To handle network errors, invalid URLs, or processing issues, the .catch() method is used to capture and manage errors that occur during the fetch process.

    Basic GET request using fetch():

    fetch('https://api.example.com/data') 
            // Replace with your API endpoint
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return response.json(); 
    // Or .text() for plain text, .blob() for binary data, etc.
  })
  .then(data => {
    console.log('API Response:', data);
    // Process the data here
  })
  .catch(error => {
    console.error('Error fetching data:', error);
  });

    Making a POST Request using fetch();

    fetch('https://api.example.com/submit-data', {

    AJAX and XMLHttpRequest

    AJAX (Asynchronous JavaScript and XML) is a technique that allows web pages to send and receive data from a server without reloading the entire page. It enables dynamic updates and smoother user experiences.

    XMLHttpRequest (XHR) is the JavaScript object traditionally used to implement AJAX. It allows JavaScript to make HTTP requests to a server and handle responses asynchronously.

    With XMLHttpRequest, a request is created, configured with a request method and URL, sent to the server, and monitored for state changes. When the response is received, JavaScript can process the returned data and update the web page dynamically.

    Although AJAX originally focused on XML, it is now commonly used with JSON data. Modern JavaScript applications often use the Fetch API instead of XMLHttpRequest, but both serve the same purpose of retrieving data asynchronously from a server.
