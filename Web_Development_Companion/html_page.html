What is HTML

Web pages are created using HyperText Markup Language (HTML), which is the authoring language used to create documents on the World Wide Web. HTML uses a set of special instructions called tags or markup to define the structure and layout of a Web document and specify how the page is displayed in a browser. every element in an html always has an opening tag and a closing tag. For example:

<b>

Not all elements consist of opening and closing tags. Some elements simply receive their content or behavior from attributes within a single tag. for example the <meta> tag has self inclosing tag like <br> and other self enclosing enclosing tags are: <embed>, <hr>, <img>, <input>, <link>, <param>, <source>, <wbr>.

Document Structure

DOCTYPE Declaration

The <!DOCTYPE html> is a modern declaration to tell the browser: "This document uses HTML5". It ensures browsers render the page in standarded mode and it also makes sure that you have it is modern. 

<!DOCTYPE html>

HTML Root Element

The <html lang="en"> tag serves as the root container for the entire HTML document. It should be the first element (after the DOCTYPE declaration) and wraps all other content on the page. 

<html lang="en">
  <All content goes here for the page>
</html>

Head Section

The <head> section contains invisible page settings and information. It defines the character encoding, viewport for mobile devices, page title, and metadata like author and description. This information helps browsers and search engines understand your page. 

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="author" content="Your name">
  <meta name="description" content="Description of the page"
  <title>Page Title</title>
</head>

Body Section

The <body> tag contains all the visible content that users see in their browser window. This includes text, images, links, buttons, and any other elements that make up the webpage interface. Everything inside <body> appears on the actual page. 

<body>
  <!-- Visible content goes here -->
</body>

Basic Text Elements

Paragraph

Any document starts with a heading. You can use different sizes for your headings. HTML provides six headings (h1 through h6) for specifying the relative importance of information. Heading element h1 is considered the most significant heading and is rendered in a larger font than the other five headings. Each successive heading element (i.e., h2, h3, etc.) is rendered in a progressively smaller font. While displaying any heading, browser adds one line before and one line after that heading. 

<p>This is a paragraph of text.</p>

Headings (h1-h6)

The <h1> through <h6> tags define HTML headings, with <h1> being the highest (most important) level and <h6> being the lowest. They create a visual and structural hierarchy for your content, similar to chapters and subchapters in a book. 

<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>
<h4>Subsection Title</h4>
<h5>Subsection Title</h5>
<h6>Subsection Title</h6>

Bold Text

The <b> tag makes text appear in bold without giving it any extra importance or semantic meaning. It's purely for visual styling. 

<b>This text is bold</b>

Strong Text

The <strong> element indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render this text in bold, but its purpose is semantic rather than purely visual. 
<strong>This text is important!</strong>

Line Break

The <br> HTML element produces a line break in text (carriage-return). It is a void element, meaning it has no content and no closing tag. Use it within paragraphs or other text containers to force text onto a new line. 

First line<br>Second line

Italic text or Emphasis

The <i> tag applies italic styling to text for purely visual purposes, while <em> (emphasis) adds semantic meaning indicating stressed emphasis. Both typically appear italicized, but <em> carries importance for screen readers and search engines, unlike <i> which is purely presentational.

<i>This text is italicized</i>
<em>Emphasized text</em>

strikethrough

The <s> HTML element renders text with a strikethrough line to indicate content that is no longer accurate or relevant.

<s>This text is no longer accurated for reading or writing</s>

Underlined Text

The <u> tag applies underlined styling to text for purely visual emphasis. Unlike semantic elements, it doesn't convey specific meaning—it's simply presentational styling to make text stand out visually.

<u>This is a paragraph</u>

Horizontal Rule

The <hr> HTML element represents a thematic break between paragraph-level elements, creating a horizontal line to visually separate content sections. It is a void element with no closing tag. 

<p>Section one content</p>
<hr>
<p>Section two content</p>

Link with Href Attribute

Attributes generally go inside the opening HTML tag, like <a href="value">. The attribute name is a predefined name from the HTML specification. There are many: href, title, class, src, etc. The attribute value is the data for the attribute, enclosed in quotes.

<a href="http://www.bmcc.cuny.edu">BMCC</a>

Block Elements vs Inline elements

One major concept to grasp in HTML is the difference between block elements and inline elements.

Block elements are those that take up the full width available on a web page, effectively blocking out any other elements from sitting next to it on the left or right.

Inline elements are those who only take up as much width as is needed to display the contents of the element, thereby allowing other elements to be in line with the inline element.

Common block elements: Paragraphs (<p>), Headers (<h1> through <h6>), Divisions (<div>), List and List Items (<ol>, <ul>, <li>), Forms (<form>)

Common inline elements: Spans (<span>), Images (<img>), Anchors (<a>)

When you're first getting into positioning and layout with HTML and CSS, it can be confusing as to why some things are placed on the screen the way they are.

Container Elements

Division (div)

The <div> HTML element is a generic container for flow content that has no inherent semantic meaning. It's commonly used with the class attribute for CSS styling and JavaScript manipulation, allowing you to group and style blocks of content together.

<div class="container">
  <p>Content inside a div</p>
</div>

Span

The <span> tag is an inline container used to mark up part of a text or document for styling or scripting. Unlike <div> which creates block-level containers, <span> stays within the flow of text, making it ideal for applying CSS classes to specific words or phrases without breaking the content layout.

<p>This is <span class="highlight">highlighted</span> text.</p>

Code Element

The <code> HTML element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code. By default, browsers render <code> text in a monospace (fixed-width) font like Courier, making it distinct from surrounding content.

Use <code>console.log()</code> for debugging.

Preformatted Text

The <pre> HTML element represents preformatted text which is to be presented exactly as written in the HTML file. Text within a <pre> element is displayed in a fixed-width font, and both spaces and line breaks are preserved, unlike regular HTML where whitespace is collapsed.

<pre>
  This text
  keeps    its
  formatting.
</pre>

List Elements

List Item

The <li> (list item) element defines an individual item inside parent list containers: <ul> for unordered (bulleted) lists or <ol> for ordered (numbered) lists. Multiple <li> elements create the actual list content, with browsers automatically adding bullets or numbers.

<li>List item content</li>

Unordered List

The <ul> HTML element represents an unordered list of items, typically rendered as a bulleted list. It serves as a container for <li> (list item) elements, grouping related items where the order doesn't matter semantically.

<ul>
  <li>First item</li>
  <li>Second item</li>
</ul>

Ordered List

The <ol> HTML element represents an ordered list of items, typically rendered as a numbered list. It contains <li> (list item) elements where the sequence matters, such as instructions, rankings, or step-by-step procedures.

<ol>
  <li>First step</li>
  <li>Second step</li>
</ol>

Description List

The <dl> HTML element represents a description list, containing groups of terms (<dt>) and descriptions (<dd>). It's commonly used for glossaries, metadata displays, or any name-value pair listings where terms need associated explanations.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

Links & Navigation

Anchor (Link)

The <a> HTML element (anchor) creates hyperlinks to other web pages, files, locations within the same page, email addresses, or any URL. The essential href attribute specifies the link destination, while the link text appears between the opening and closing tags.

<a href="page.html">Click here</a>

External Link with Target

The target="_blank" attribute in an <a> tag opens the linked document in a new browser tab or window. This keeps your original page open while allowing users to explore external content, improving navigation for external references, documentation, or resources.

<a href="https://example.com" target="_blank">Visit Example</a>

Email Link

The mailto: protocol in an <a> tag's href attribute creates a link that opens the user's default email client with a new message. When clicked, it pre-fills the recipient address, and can optionally include subject, body, and CC/BCC fields using URL parameters.

<a href="mailto:email@example.com">Send Email</a>

Internal Bookmarks

Internal bookmarks use fragment identifiers (#) in <a> tags to link to specific sections within the same page. The href="#id" points to an element with a matching id attribute, allowing users to jump directly to that location without scrolling manually.

<a href="#section1">Go to Section 1</a>
...
<h2 id="section1">Section 1</h2>

Images & Media

Image Element

The <img> HTML element embeds an image into the document. It requires two essential attributes: src (source) specifies the image file path or URL, while alt (alternative text) provides descriptive text for accessibility, screen readers, and when images cannot load.

<img src="image.jpg" alt="Description of image">

Image as Link

Wrapping an <img> element inside an <a> tag creates a clickable image link. The image acts as the link's visual content, with the href attribute determining the destination. Always include descriptive alt text for accessibility since the image serves as interactive content.

<a href="page.html"><img src="button.jpg" alt="Click me"></a>

HTML Entities (Special Characters)

Non-breaking Space

The &nbsp; HTML entity (non-breaking space) creates a space character that prevents browsers from breaking a line at that position. It ensures that connected words or values (like "$10 USD") stay together on the same line, maintaining readability and preventing awkward line breaks.

Price: $10 USD

Less Than & Greater Than

The &lt; and &gt; HTML entities allow you to display the literal characters < and > without browsers interpreting them as HTML tags. Use &lt; for "less than" (<) and &gt; for "greater than" (>) when displaying code examples or mathematical expressions.

< for < and > for >

Ampersand

The &amp; HTML entity displays the ampersand symbol (&) without browsers interpreting it as the start of another character entity. Since & begins all HTML entities, you must use &amp; when you want to show a literal ampersand character in your content.

Company & Co.

Copyright & Trademark

The &copy; and &reg; HTML entities display copyright (©) and registered trademark (®) symbols. Use &copy; for copyright notices like "© 2024 Company" and &reg; for registered trademarks, ensuring proper legal symbols appear consistently across all browsers and devices.

© 2024 & ® BrandName

Currency Symbols

HTML entities like &dollar;, &euro;, and &pound; display currency symbols consistently across different systems. Use &dollar; for dollar ($), &euro; for euro (€), and &pound; for pound sterling (£) to ensure currency symbols render properly regardless of the user's language or font settings. 

Price: $100 & €90 & £80

External Resources

Linking CSS

The <link> element with rel="stylesheet" connects an external CSS file to an HTML document, typically placed inside the <head> section. The href attribute specifies the path to the CSS file, allowing you to separate styling from HTML structure for cleaner, more maintainable code.

<link rel="stylesheet" href="styles.css">

Linking Bootstrap

The <link> element connects the Bootstrap CSS framework to your HTML document, providing pre-designed responsive components and layout systems. By linking bootstrap.min.css (the minified version), you enable Bootstrap's grid system, components, and utilities while ensuring faster loading times.

<link rel="stylesheet" href="bootstrap.min.css">

Script

The <script> HTML element is used to embed or reference executable JavaScript code. When including the src attribute like <script src="yourscriptfile.js"></script>, it links to an external JavaScript file, typically placed before the closing </body> tag for better page loading performance.

<script src="yourscriptfile.js"></script>

HTML 5 Structure elements

Semantic HTML elements provide meaningful structure to web documents, helping both browsers and developers understand content organization. The <section> element serves as a generic container for thematic groupings of content, often used with heading tags (<h1>-<h6>) to define document hierarchy. Within sections, <article> encapsulates self-contained, independently distributable content like blog posts or news stories. The <aside> element houses supplementary material that's tangentially related to the main content, such as sidebars or pull quotes. Each section typically features a <header> at its beginning for introductory content and navigation, with <nav> specifically designating navigation link collections. For multimedia content, <figure> pairs embedded media like images or videos with captions using <figcaption>. Conversations and dialog interfaces can be marked up using the <dialog> element. Finally, <footer> elements conclude sections with metadata like author information, copyright notices, or related links. Together, these semantic tags create a clearly defined, accessible document structure that improves SEO, screen reader compatibility, and code maintainability compared to generic <div> containers. 

<section class="main-content">
  <header>
    <h1>My Awesome Blog</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
      <a href="#contact">Contact</a>
    </nav>
  </header>

  <article>
    <h2>Understanding Semantic HTML</h2>
    <p>Semantic elements clearly describe their meaning...</p>
    
    <figure>
      <img src="semantic-html.png" alt="Semantic HTML structure">
      <figcaption>Figure 1: HTML5 Semantic Layout</figcaption>
    </figure>
    
    <aside>
      <h3>Did You Know?</h3>
      <p>HTML5 was published in 2014!</p>
    </aside>

        <p>Add your comment below:</p>
        <textarea></textarea>

      </form>

  </footer>
</section>

Advanced Features

Nested Lists

Nested lists are created by placing a complete <ul> or <ol> list inside an <li> element of a parent list. This creates hierarchical structures like outlines, menus, or category-subcategory relationships, with each nested level typically displaying different indentation and bullet/number styles. 

<ul>
  <li>Category
    <ul>
      <li>Sub-item 1</li>
      <li>Sub-item 2</li>
    </ul>
  </li>
</ul>

List Types

The type attribute on <ol> and <ul> elements controls list marker styling. For ordered lists, type="A" uses uppercase letters, type="I" uses Roman numerals. For unordered lists, type="square" uses square bullets (though CSS list-style-type is now preferred for styling flexibility).

<ol type="A"> <A, B, C>
<ol type="I"> <I, II, III>
<ul type="square"> <square bullets>

Forms

HTML Forms enable user interaction by collecting data through elements like text fields, buttons, checkboxes, and dropdowns. This data can be sent to servers or processed on the page itself. Each form element should be paired with descriptive labels for accessibility and clear user guidance. From a user experience (UX) point of view, it's important to remember that the bigger your form, the more you risk losing users. Keep it simple and stay focused: ask only for that data you absolutely need.

Form Element

The <form> element defines a container for interactive controls that collect user data. All form elements must be placed inside a <form> tag, but forms cannot be nested within each other. While HTML5 allows form elements outside forms using the form attribute, browser support for this feature is currently inconsistent.

Form elements are different types of input elements, like text fields, checkboxes, radio buttons, select drop-downs, submit buttons and more.

<form action="my-handling-form-page" method="post">
  <!-- form elements -->
</form>

Form element Attributes

Form attributes control submission behavior: action specifies where data is sent, target determines where the response opens (new tab/window), and method defines the HTTP method (GET for visible parameters or POST for secure submission). GET: Data appears in URL, limited length, good for bookmarks, never for sensitive data. POST: Data hidden, no length limits, cannot bookmark, use for sensitive information.

<form action="action_page.php" target="_blank" method="get">
  <input type="text" name="search">
  <button type="submit">Search</button>
</form>
<form action="action_page.php" target="_blank" method="post">
  <input type="text" name="search">
  <button type="submit">Search</button>
</form>

Name attribute of input elements

Input fields require a name attribute for their data to be submitted. Without it, the field's data is ignored and not sent to the server.

The input element is the primary form control, with its behavior determined by the type attribute: text for single-line input, radio for single selection, and submit for form submission.

Text Input

<form>
  First Name:<br>
  <input type="text" name="firstname"><br>
  Last Name:<br>
  <input type="text" name="lastname"><br>
</form>

Input Radio Button Input

Radio buttons allow single selection from multiple options, with only one choice possible at a time.

<form>
  <input type="radio" name="gender" value="male" checked> Male<br>
  <input type="radio" name="gender" value="female"> Female<br>
  <input type="radio" name="gender" value="other"> Other<br>
</form>

Input Type Submit

The submit button sends form data to the specified form-handler, which is typically a server script defined in the form's action attribute.

<form>
  First Name: <input type="text" name="firstname"><br>
  Last Name: <input type="text" name="lastname"><br>
  <input type="submit" value="Submit">
</form>

Input Type Password

Password fields hide entered text with dots or asterisks for security.

<form>
  User Name:<br>
  <input type="text" name="username"><br>
  User Password:<br>
  <input type="password" name="psw"><br>
</form>

Select Element

The <select> element defines a drop-down list:

<select name="cars">
    <option value="volvo">Volvo</option>
    <option value="saab">Saab</option>
    <option value="fiat">Fiat</option>
    <option value="audi">Audi</option>
</select>

Option element

The option tag creates selectable items in dropdowns. The first option is selected by default, or use the selected attribute to preselect any option.

<select name="cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
  <option value="fiat" selected>Fiat</option>
  <option value="audi">Audi</option>
</select>

Select 'size' (increasing visible values)

Use the `size` attribute to specify the number of visible values:

<select name="cars" size="2">
    <option value="volvo">Volvo</option>
    <option value="saab">Saab</option>
    <option value="fiat">Fiat</option>
    <option value="audi">Audi</option>
</select>

Textarea

The <textarea> element defines a multi-line input field (a text area)

The rows attribute specifies the visible number of lines in a text area. The cols attribute specifies the visible width of a text area. 

<textarea name="message" rows="10" cols="30">
The cat was playing in the garden.
</textarea>

Button Element

The <button> element defines a clickable button:

<button type="button" onclick="alert('Hello World!')">

</button>

New Html 5 elements

HTML5 added datalist/output elements. Older browsers safely ignore unsupported tags.

Datalist Element

The datalist provides predefined suggestions for input fields through a dropdown, linked via matching id and list attributes.

<form action="result.php">

  <datalist id="browsers">
    <option value="Internet Explorer">
    <option value="Firefox">
    <option value="Chrome">
    <option value="Opera">
    <option value="Safari">
  </datalist>
</form>
